# Research

Library references, API evaluations, and technology research.

## How Notes Get Here

Run `/research <library-or-topic>` — the research command:
1. Checks for existing note (<14 days old = reuse)
2. WebSearches for authoritative docs
3. Distills to <100-line reference note
4. Saves here with today's date

## Note Format

Each research note should include:
- **Version** — exact version researched
- **Key APIs** — functions/classes used in this project
- **Gotchas** — breaking changes, common mistakes, edge cases
- **Sources** — URLs to official docs

## Freshness

- Notes >14 days old should be re-verified before relying on them
- VaultKeeper flags stale notes
- `INDEX.md` tracks all research notes with their freshness status

## Owner

Any agent via `/research`. VaultKeeper audits freshness.

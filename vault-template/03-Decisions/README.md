# Decisions

Architecture Decision Records (ADRs) for every non-obvious design choice.

## ADR Format

```
# ADR-NNN: [Title]
Date: [YYYY-MM-DD]
Status: [PROPOSED | ACCEPTED | DEPRECATED | SUPERSEDED by ADR-XXX]

## Context
[What forced this decision]

## Decision
[What we decided]

## Consequences
[What this makes easier and harder]

## Alternatives Considered
[What we rejected and why]
```

## Rules

- Every non-trivial design decision gets an ADR
- ADRs are never deleted — only DEPRECATED or SUPERSEDED
- Reference ADRs from DISPATCH.md tasks and implementation code
- Architect owns creation; Reviewer checks compliance

## Owner

Architect agent.

# Patterns

Code patterns in use across the project. Documents HOW things are built.

## Expected Notes

- Repository pattern, service layer pattern, etc.
- Error handling patterns
- Testing patterns
- Authentication/authorization patterns
- Caching patterns
- Logging/observability patterns

## Rules

- When Reviewer identifies a pattern in use, it gets documented here
- New code should follow existing patterns (or write an ADR to change them)
- Each pattern note includes a concrete code example

## Owner

Reviewer identifies. DocsAgent documents.

## Template

Use `vault/_meta/note-template.md` with Type: Pattern.

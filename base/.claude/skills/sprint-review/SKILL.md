# Sprint Review Skill

Generate a comprehensive sprint review:

1. **Task Analysis**: Read DISPATCH.md — count completed vs planned tasks
2. **Velocity**: Compare to previous sprints if data exists
3. **Blocker Analysis**: Read BOARD.md — identify what blocked progress and for how long
4. **Quality Metrics**: Test pass rate, coverage, security scan results
5. **Scope Changes**: Identify any tasks added/removed mid-sprint
6. **Retrospective Questions**:
   - What went well?
   - What could improve?
   - What should we stop doing?
   - What action items for next sprint?

Write output to vault/09-Retrospectives/ or docs/retrospectives/.
